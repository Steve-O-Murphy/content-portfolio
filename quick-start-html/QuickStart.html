Getting Started

This information provides the basic steps for uploading content/objects to Origin Storage.

Using Origin Storage is simple. Follow 4 easy steps to get started:

1. Choose your ingest method
2. Choose your Replication and Location Policies
3. Upload Content
4. Retrieve or Share Content

1. Choose your ingest method

There are multiple ways to ingest content using Limelight Storage. Choose from the options below:

  • Orchestrate Storage: This is the easiest and most expeditious way to get started. If you don't like using a command line, this is for you.
    • Go to: https://storage.limelight.com
    • For login use your shortname/account info received in your Limelight Welcome Letter.
  • FTP, SFTP, SCP, Rsync. You can use these methods directly from the command line. Please see the Overview or Best Practices guides for more information.
  • Java FTP and API client (including Multipart support): https://github.com/Orchestrate-Cloud-Storage/StorageClient
  • APIs: Use directly from your application. Refer to the API Guide. Includes Multipart support.
  • Aspera FASP high speed file transfer is supported.

Whenever possible we recommend that customers leverage the Application Programming Interface (API) due to its feature-rich ingest workflow and its superior performance.

 

2. Choose your Replication and Location Policies

Origin Storage is pre-configured with replication policies attached to your directories that determine the locations where you content will be stored.

  • The standard policies provide geographic distribution and redundancy across the applicable region or regions.
  • The standard weighted policies provide geographic distribution in the primary region and at least one copy of the file in the secondary region.
  • The extended policy provides geographic distribution in EACH of the individual regions (US, EU, APAC).
  • The infrequent access policies provide full availability and durability in a single region and POP, for a lower cost storage option in use cases that are not as performance sensitive.

When content is uploaded to a directory, objects will be distributed according to the directory and its associated policy.

You cannot assign policies yourself. Please contact your Account Manager for assistance.

You can create directories using the or vie the APIs:

  • makeDir
  • makeDir2
  • /post/directory

Any directories you create will inherit the characteristics (and policies) of the directory in which they are created

In the Orchestrate Storage Portal, if you drag and drop a file from one directory to another, then the transferred file retains its original policy (the policy assigned to it by its original directory). However, the transferred file's URL changes to reflect the new directory. Note also that it can take a bit of time before all replication policies are applied to the transferred file

Customer Portal Workspace

If you create your own directory from scratch at the root, it will not inherit any of the pre-defined policies. Instead, it will inherit system default policies. Please see the Origin Storage Overview Guide for further details.

3. Upload Content

To upload content using the Origin Storage Customer Portal:

  • Simply drag and drop files from Windows Explorer or Mac Finder to the bottom pane of the Origin Storage Customer Portal titled “Upload Queue” within the workspace (see figure below). Before you do so, be sure you have selected the Directory in the left hand pane, titled “Folder Browser”, to which you would like the file copied.Or, alternatively, choose the “Add Files” option from the Upload Queue window at bottom, and follow the prompts.
  • Alternatively, choose the “Add Files” option from the Upload Queue window at bottom, and follow the prompts.

You may want to review the section 2. Choose your Replication and Location Policies, before you begin copying files, as it will help you decide where you would like files to reside in.

If you would like to upload content using another method (API, FTP, etc.), see the links to the guides in 1. Choose your ingest method.

4. Retrieve or Share Content

You have several options for retrieving and sharing content:

  • Use the Origin Storoage Customer Portal
    • To retrieve content, open the Origin Storage Customer Portal and click the downward arrow to the left of the file name that you wish to download:

    • To share content via email, right-click on any file name and choose the "Share Link" option:

From the Share Link dialog (below) you can cut and paste the URL into an email to share the file.

  • To download the content via a CDN, consult your CDN provider. Origin Storage can act as origin storage for any CDN.
